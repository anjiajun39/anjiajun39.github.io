如何使用Git

本文最后更新于：2020年11月20日 上午

本文记录本人学习Git相关操作

教程来源于https://oschina.gitee.io/learn-git-branching/
想学习git基本命令的小伙伴可以去找个网站，有图形树便于理解。

提交记录

git commit

创建分支

git branch <分支名>

切换分支

git checkout <分支名>

可使用以下命令直接创建并切换分支

git checkout -b <分支名>

分支合并

该命令使<分支名>合并至当前所在分支

git merge <分支名>

例如将master分支合并到bugfix
git checkout bugfix
git merge master

Git rebase

该命令把<分支名>直接移至当前分支
假设当前在bugfix分支

git rebase master

这时master分支还没有更新，我们再切换回master分支，把它rebase到bugfix分支上
git rebase bugfix

在提交树上移动,分离head

该命令使head指向cx分支，可使用哈希值

git checkout cx

相对引用

首先看看操作符 (^)。把这个符号加在引用名称的后面，表示让 Git 寻找指定提交记录的父提交。
所以 ‘master^’ 相当于“’master’的父节点”。
‘master^^’是’master’的第二个父节点
现在咱们切换到’master’的父节点

git checkout master^

“~”操作符

如果你想在提交树中向上移动很多步的话，敲那么多 ^ 貌似也挺烦人的，Git 当然也考虑到了这一点，于是又引入了操作符 ~。
该操作符后面可以跟一个数字（可选，不跟数字时与 ^ 相同，向上移动一次），指定向上移动多少次。
使用’~’一次后退四步

git checkout HEAD~4

强制修改分支位置
直接使用 -f 选项让分支指向另一个提交。例如:
git branch -f master HEAD~3

上面的命令会将 master 分支强制指向 HEAD 的第 3 级父提交。

撤销变更

Git Reset
git reset 通过把分支记录回退几个提交记录来实现撤销改动。你可以将这想象成“改写历史”。git reset 向上移动分支，原来指向的提交记录就跟从来没有提交过一样。

git reset HEAD~1

Git 把 master 分支移回到 C1；现在我们的本地代码库根本就不知道有 C2 这个提交了。
（注：在reset后， C2 所做的变更还在，但是处于未加入暂存区状态。）

Git Revert
虽然在你的本地分支中使用 git reset 很方便，但是这种“改写历史”的方法对大家一起使用的远程分支是无效的哦！
为了撤销更改并分享给别人，我们需要使用 git revert

git revert HEAD

本质上这是新提交记录了 C2’ 引入了更改 —— 这些更改刚好是用来撤销 C2 这个提交的。也就是说 C2’ 的状态与 C1 是相同的。
revert 之后就可以把你的更改推送到远程仓库与别人分享啦。
